# yandex/ — автономный сжатый билд

> ⚠️ Этот каталог **не относится к основному проекту**. Разработка игры идёт в
> `src/` и `public/` как обычно — здесь ничего трогать не нужно. Это отдельный
> инструмент, который берёт готовый `vite build` и делает из него лёгкий билд
> для загрузки на Яндекс.Игры (лимит 100 МБ) или для раздачи со своего сервера.
> Можно удалить целиком без последствий для игры.

## Запуск

```bash
cd yandex
npm install        # один раз (ставит sharp)
npm run build      # → yandex/build/ + yandex/snowfall-yandex.zip
```

## Что делает

1. `vite build --base=./` в `yandex/build/` (относительные пути → работает и в
   iframe-под-пути Яндекса, и на своём сервере).
2. Релятивизирует жёсткие `/models` `/textures` в собранном бандле
   (в исходниках они абсолютные — правится только копия в `build/`, `src/` цел).
3. Каждую `.gltf`-модель: текстуры → **WebP** (`WEBP_Q`, дефолт 80),
   геометрию → **quantize** (`KHR_mesh_quantization`, three понимает нативно).
4. Текстуры земли (снег) → пережатый `.jpg` с тем же именем.
5. Пакует в `snowfall-yandex.zip`.

**Почему не Draco и не `optimize`:** Draco потребовал бы `DRACOLoader` в рантайме
(правка `src/`), а `optimize`/`join`/`flatten`/`weld` переименовывают меши и
сломали бы матчинг по именам в `src/trees.js` (`Pine_*_LOD*`, `bark`, `clusters`).
Здесь только `webp` + `quantize` — оба безопасны и не требуют правок игры.

## Ручки

| env | эффект |
|-----|--------|
| `WEBP_Q=90` | выше качество текстур (крупнее билд) |
| `WEBP_Q=70` | мельче билд, заметнее артефакты |
| `NO_QUANTIZE=1` | не квантовать геометрию (если увидишь артефакты на моделях) |
| `NO_ZIP=1` | не паковать zip |

## Проверить локально

```bash
npx serve yandex/build      # открыть в браузере, поиграть
```
